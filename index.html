Pasos para levantar tu primera página web con Django desde cero

📝 Los nombres como nombre_entorno, nombre_proyecto y nombre_app son ejemplos genéricos — tú los eliges al crear tu proyecto. Solo úsalos de forma consistente en todos los pasos.

01 — Preparación del entorno
1

Crear el entorno virtual

python -m venv nombre_entorno
Crea una carpeta con el nombre que elijas — en el ejemplo del profesor fue ambiente_tres, pero puede ser cualquier nombre como env o venv. Esta carpeta aísla las librerías de tu proyecto del resto de la computadora. Hazlo una sola vez por proyecto.
2

Activar el entorno virtual

source nombre_entorno/Scripts/activate
Reemplaza nombre_entorno con el nombre que usaste en el paso 1. Debes activarlo cada vez que abras una terminal nueva. Sabrás que está activo porque verás el nombre entre paréntesis al inicio: (nombre_entorno).
⚠️ En Windows usa Scripts\activate (con backslash). En Mac/Linux usa bin/activate.
3

Instalar Django

pip install django
Este comando es siempre igual. Solo se instala dentro del entorno virtual activo — si no lo activaste primero, Django se instala globalmente y puede causar conflictos.
02 — Crear el proyecto y la app
4

Crear el proyecto Django

django-admin startproject nombre_proyecto
Elige el nombre de tu proyecto — en el ejemplo del profesor fue demo3. Genera una carpeta con ese nombre que contiene los archivos de configuración: settings.py, urls.py, wsgi.py, etc.
5

Entrar a la carpeta del proyecto

cd nombre_proyecto
⚠️ Este paso es obligatorio. Usa el mismo nombre que pusiste en el paso 4. Si no entras a la carpeta, los comandos de Django no funcionarán correctamente.
6

Crear la app dentro del proyecto

python manage.py startapp nombre_app
Elige el nombre de tu app — en el ejemplo fue miapp. Una "app" es un módulo dentro del proyecto; el proyecto puede tener varias. Crea una carpeta con ese nombre que contiene views.py, models.py, etc.
7

Aplicar migraciones de la base de datos

python manage.py migrate
Este comando es siempre igual. Django crea las tablas necesarias en db.sqlite3. Corre esto antes de levantar el servidor por primera vez.
8

Crear el superusuario (admin)

python manage.py createsuperuser
Este comando es siempre igual. Te pedirá nombre de usuario, correo y contraseña. Con este usuario puedes entrar al panel de administración en /admin/.
9

Abrir el proyecto en VS Code

code .
El punto . significa "la carpeta actual". Abre todo el proyecto en VS Code para que puedas editar los archivos.
03 — Configurar el proyecto 📁 nombre_proyecto/
10

Registrar la app en nombre_proyecto/settings.py

Abre settings.py dentro de la carpeta del proyecto (no de la app). Busca INSTALLED_APPS y agrega tu app al final. El formato es nombre_app.apps.NombreAppConfig — la primera parte en minúsculas, la segunda con inicial mayúscula: 
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    ...
    'nombre_app.apps.NombreAppConfig',  # ← agrega esto
]
⚠️ Ejemplo: si tu app se llama miapp, escribes 'miapp.apps.MiappConfig'. Puedes verificar el nombre exacto de la clase abriendo el archivo nombre_app/apps.py.
13

Conectar las URLs de la app en nombre_proyecto/urls.py

Abre el urls.py de la carpeta del proyecto. Agrega include a la importación y una entrada en urlpatterns que apunte a las URLs de tu app: 
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('nombre_app.urls')),  # ← agrega esto
]
14

Crear carpetas static/ y templates/

Estas carpetas van en la raíz del proyecto, al mismo nivel que manage.py, no dentro de la app. Créalas manualmente desde VS Code. 
nombre_proyecto/      # raíz — donde está manage.py
├── templates/           # aquí van los archivos HTML
│   └── index.html
├── static/              # aquí van CSS, imágenes, JS
│   ├── css/
│   └── img/
├── manage.py
├── nombre_proyecto/    # carpeta de configuración
└── nombre_app/         # carpeta de la app
15

Configurar rutas de templates y static en settings.py

Asegúrate de tener import os al inicio del archivo. Luego edita estas dos secciones — el código es siempre igual, sin importar el nombre de tu proyecto: 
# Al inicio del archivo
import os

# En TEMPLATES, modifica el valor de DIRS:
TEMPLATES = [{
    ...
    'DIRS': [os.path.join(BASE_DIR, 'templates')],
    ...
}]

# Al final del archivo, después de STATIC_URL:
STATIC_URL = '/static/'
STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static')]
⚠️ Sin esto, Django no encontrará tus HTMLs ni archivos CSS/imágenes aunque las carpetas existan.
04 — Configurar la app 📁 nombre_app/
11

Crear la vista en nombre_app/views.py

Abre views.py dentro de la carpeta de tu app y agrega una función. El nombre de la función lo decides tú: 
from django.shortcuts import render

def nombre_vista(request):
    return render(request, "nombre_archivo.html")
⚠️ El nombre del HTML que escribas aquí debe coincidir exactamente con el archivo que pongas en la carpeta templates/.
12

Crear nombre_app/urls.py y definir rutas

⚠️ Este archivo no existe por defecto — debes crearlo manualmente dentro de la carpeta de tu app. 
from django.urls import path
from . import views

urlpatterns = [
    path('', views.nombre_vista, name='inicio'),
]
from . import views significa "importa views del mismo directorio (la app)". Reemplaza nombre_vista con el nombre de la función que definiste en views.py.
05 — El archivo HTML con archivos estáticos 📁 templates/
16

Estructura de templates/nombre_archivo.html

La primera línea siempre debe ser {% load static %}, antes del <!DOCTYPE>. Luego usas {% static 'ruta' %} para cargar tus archivos. La ruta parte desde adentro de la carpeta static/: 
{% load static %}  ← SIEMPRE en la primera línea

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Mi página</title>

  <!-- Cargar CSS -->
  <link rel="stylesheet" href="{% static 'css/archivo.css' %}">

</head>
<body>

  <!-- Cargar una imagen -->
  <img src="{% static 'img/imagen.jpg' %}" alt="descripción">

  <!-- Cargar JS (si tienes) -->
  <script src="{% static 'js/archivo.js' %}"></script>

</body>
</html>
⚠️ Los nombres de archivos deben coincidir exactamente con los que tienes en static/. Las mayúsculas importan: Imagen.jpg ≠ imagen.jpg.
!

¿Por qué usar {% static %} y no rutas directas?

❌ MAL — ruta escrita a mano, puede romperse:
<link href="/static/css/estilos.css">

✅ BIEN — Django construye la ruta automáticamente:
<link href="{% static 'css/estilos.css' %}">
Con {% static %}, si cambias STATIC_URL en settings.py, todas las rutas se actualizan solas.
06 — Verificar que todo funciona
✓

Levantar el servidor de desarrollo

python manage.py runserver
✅ Abre tu navegador en http://127.0.0.1:8000/ — si ves tu página HTML, todo está configurado correctamente. Para detener el servidor presiona Ctrl + C.

07 — Referencia rápida: etiquetas de plantilla Django
{% load static %}
Carga el módulo de archivos estáticos. Va una sola vez, en la primera línea.
{% static 'ruta' %}
Genera la URL de un archivo estático (CSS, imagen, JS).
{{ variable }}
Muestra el valor de una variable enviada desde views.py.
{% if %} {% endif %}
Condicional dentro del HTML.
{% for %} {% endfor %}
Ciclo para repetir elementos HTML.
{% block %} {% endblock %}
Para herencia de plantillas (temas más avanzados).
