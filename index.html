Pasos para levantar tu primera página web con Django desde cero

📝 Los nombres en naranja como nombre_entorno, nombre_proyecto o nombre_app son los que tú decides al crear tu proyecto. Lo importante es usar siempre el mismo nombre en todos los pasos.

01 — Preparación del entorno
1
📍 Terminal — desde cualquier ubicación

Crear el entorno virtual

python -m venv nombre_entorno
Esto crea una carpeta con el nombre que elijas, donde se van a instalar las librerías de este proyecto. Se hace una sola vez por proyecto.
2
📍 Terminal — misma ubicación del paso anterior

Activar el entorno virtual

source nombre_entorno/Scripts/activate
Le indicamos a Python que use el entorno que acabamos de crear. Cuando está activo, verás el nombre entre paréntesis al inicio de la terminal: (nombre_entorno).
⚠️ Hay que activarlo cada vez que abras una terminal nueva.
Windows: Scripts\activate  |  Mac/Linux: bin/activate
3
📍 Terminal — con el entorno virtual activo

Instalar Django

pip install django
Descargamos e instalamos Django dentro del entorno que activamos. Este comando siempre es igual.
02 — Crear el proyecto y la app
4
📍 Terminal — con el entorno virtual activo

Crear el proyecto Django

django-admin startproject nombre_proyecto
Django genera automáticamente la estructura base del proyecto: una carpeta con el nombre que le demos, con los archivos de configuración general adentro — settings.py, urls.py, wsgi.py, entre otros.
5
📍 Terminal — justo después de crear el proyecto

Entrar a la carpeta del proyecto

cd nombre_proyecto
Nos movemos dentro de la carpeta del proyecto. Todos los comandos que siguen deben ejecutarse desde aquí adentro, donde está el archivo manage.py.
⚠️ Si no entras a esta carpeta antes de continuar, los siguientes comandos van a fallar.
6
📍 Terminal — dentro de la carpeta del proyecto

Crear la app

python manage.py startapp nombre_app
Creamos la app, que es una subcarpeta dentro del proyecto donde vamos a escribir el código de nuestra página — las vistas, los modelos, las rutas propias. Django genera los archivos base por nosotros.
7
📍 Terminal — dentro de la carpeta del proyecto

Crear las tablas base de la base de datos

python manage.py migrate
Django necesita ciertas tablas en la base de datos para funcionar — para usuarios, sesiones y otras cosas internas. Este comando las crea en un archivo llamado db.sqlite3. Siempre se corre antes de levantar el servidor por primera vez.
8
📍 Terminal — dentro de la carpeta del proyecto

Crear el usuario administrador

python manage.py createsuperuser
La terminal nos va a pedir un nombre de usuario, correo y contraseña. Con esos datos se crea el usuario con el que podemos entrar al panel de administración de Django en /admin/.
9
📍 Terminal — dentro de la carpeta del proyecto

Abrir el proyecto en VS Code

code .
Abrimos VS Code apuntando a la carpeta actual. El punto significa "aquí mismo". A partir de este paso vamos a editar archivos, así que abrimos el editor.
03 — Configurar el proyecto 📁 carpeta del proyecto
10
📂 nombre_proyecto / settings.py

Registrar la app en la configuración del proyecto

Le indicamos al proyecto que existe nuestra app y que la incluya. Si no la registramos aquí, Django la ignora. Buscamos la lista INSTALLED_APPS y agregamos una línea al final: 
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    ...  # las líneas que ya están
    'nombre_app.apps.NombreAppConfig',  # ← agregamos esto
]
⚠️ El formato es: el nombre de la app en minúsculas, luego .apps., luego el mismo nombre con inicial mayúscula y al final Config. Puedes ver el nombre exacto abriendo nombre_app/apps.py.
13
📂 nombre_proyecto / urls.py

Conectar las rutas de la app al proyecto

Le decimos al proyecto que las rutas de nuestra app están en el archivo urls.py de la app. Agregamos include en la importación y una línea en urlpatterns: 
from django.urls import path, include  # agregar "include"

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('nombre_app.urls')),  # ← agregamos esto
]
14
📂 Raíz del proyecto — al mismo nivel que manage.py

Crear las carpetas templates/ y static/

Creamos dos carpetas manualmente desde VS Code. En templates/ van a vivir los archivos HTML, y en static/ van el CSS, las imágenes y el JavaScript. 
nombre_proyecto/         ← aquí está manage.py
├── templates/             ← aquí van los HTML
│   └── index.html
├── static/                ← aquí van CSS, imágenes, JS
│   ├── css/
│   └── img/
├── manage.py
├── nombre_proyecto/       ← carpeta de configuración
└── nombre_app/            ← carpeta de la app
⚠️ Estas dos carpetas van en la raíz del proyecto, no dentro de la carpeta de la app.
15
📂 nombre_proyecto / settings.py

Indicarle a Django dónde están los templates y los archivos estáticos

Le decimos a Django en qué carpetas tiene que buscar los archivos HTML y los archivos estáticos. Primero importamos os al inicio del archivo, luego modificamos dos secciones: 
# Al inicio del archivo:
import os

# En TEMPLATES, le indicamos dónde buscar los HTML:
TEMPLATES = [{
    ...
    'DIRS': [os.path.join(BASE_DIR, 'templates')],
    ...
}]

# Al final del archivo, le decimos de dónde sacar los estáticos:
STATIC_URL = '/static/'
STATICFILES_DIRS = [os.path.join(BASE_DIR, 'static')]
⚠️ Sin esto, Django no va a encontrar ni los HTML ni el CSS ni las imágenes, aunque las carpetas existan.
Este código es siempre igual — no cambia según el nombre de tu proyecto. BASE_DIR es una variable que Django ya tiene definida y apunta a la raíz del proyecto.
04 — Configurar la app 📁 carpeta de la app
11
📂 nombre_app / views.py

Crear la función que va a mostrar la página

En Django, una vista es una función que recibe la petición del navegador y le responde con un archivo HTML. La escribimos en views.py, que está dentro de la carpeta de la app: 
from django.shortcuts import render

def nombre_vista(request):
    return render(request, "nombre_archivo.html")
⚠️ El nombre del HTML que escribas aquí tiene que coincidir exactamente con el archivo que pongas en la carpeta templates/.
12
📂 nombre_app / urls.py — hay que crearlo manualmente

Crear el archivo de rutas de la app

Le indicamos a Django qué función debe ejecutar cuando alguien entre a una dirección. Este archivo no existe por defecto — hay que crearlo dentro de la carpeta de la app. Una vez creado, escribimos: 
from django.urls import path
from . import views  # el punto indica que views está en esta misma carpeta

urlpatterns = [
    path('', views.nombre_vista, name='inicio'),
]
path('', ...) con comillas vacías corresponde a la página principal, es decir /. El nombre de la vista tiene que ser el mismo que definiste en views.py.
05 — El archivo HTML 📁 templates/
16
📂 templates / nombre_archivo.html

Escribir el HTML y cargar los archivos estáticos

Para que Django pueda servir el CSS y las imágenes desde la carpeta static/, usamos la etiqueta {% static %}. La primera línea del archivo siempre tiene que ser {% load static %} — eso le dice a Django que vamos a usar esas etiquetas: 
{% load static %}  ← activa las etiquetas de Django. Va antes del DOCTYPE

<!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Mi página</title>

  <!-- Le decimos de dónde sacar el CSS -->
  <link rel="stylesheet" href="{% static 'css/archivo.css' %}">

</head>
<body>

  <!-- Le decimos de dónde sacar la imagen -->
  <img src="{% static 'img/imagen.jpg' %}" alt="descripción">

  <!-- Le decimos de dónde sacar el JS -->
  <script src="{% static 'js/archivo.js' %}"></script>

</body>
</html>
⚠️ La ruta dentro de {% static '...' %} empieza desde adentro de la carpeta static/. Si tu imagen está en static/img/foto.jpg, escribes {% static 'img/foto.jpg' %}.
!
💡 ¿Por qué usar {% static %} en lugar de escribir la ruta directo?

La diferencia entre hacerlo bien y hacerlo mal

❌ Así NO — Django no puede gestionar esta ruta:
<link href="/static/css/estilos.css">

✅ Así SÍ — Django genera la ruta correcta automáticamente:
<link href="{% static 'css/estilos.css' %}">
Con {% static %}, Django construye la ruta hacia el archivo. Si en algún momento cambia la configuración de los estáticos, el HTML no hay que tocarlo.
06 — Levantar el servidor y verificar
✓
📍 Terminal — dentro de la carpeta del proyecto, con el entorno activo

Levantar el servidor de desarrollo

python manage.py runserver
Iniciamos el servidor local. Abrimos el navegador y entramos a http://127.0.0.1:8000/ — si todo está configurado correctamente, vemos nuestra página. Para detener el servidor presionamos Ctrl + C.
✅ Si ves tu página, todo está funcionando correctamente.

07 — Referencia rápida: etiquetas de plantilla Django
{% load static %}
Activa las etiquetas de Django en el HTML. Va una sola vez, en la primera línea, antes del DOCTYPE.
{% static 'ruta' %}
Le dice a Django de dónde sacar un archivo estático. La ruta parte desde dentro de la carpeta static/.
{{ variable }}
Muestra en el HTML el valor de una variable que envió views.py desde Python.
{% if %} {% endif %}
Muestra o esconde partes del HTML según una condición.
{% for %} {% endfor %}
Repite un bloque de HTML por cada elemento de una lista.
{% block %} {% endblock %}
Permite reutilizar una plantilla base y reemplazar solo secciones específicas.
